JJ Keith

Hello!

I'm a front end developer with a love for clean styles, responsive design, great documentation, and learning new technologies. Before I got into web development, I was a writer and an educator, which basically means that I've shifted from talking too much about the Great Oxford Comma debate to discussing CSS nomenclature systems or comparing the merits of rem versus vw/vh.

I've been a full-time web-developer for just over a year, but my experience with HTML and CSS goes back so far that I dare not put a time frame on it. Suffice to say, Geocities was involved.

Since adding JavaScript to my toolkit, I've been learning new stuff like crazy. With my current employer, Gigmor, a Santa Monica startup that connects musicians with venues and bookers, I've been primarily using HTML/JSX, Sass, Foundation, JavaScript and React, with some cameos from PHP, Laravel, JQuery and Bootstrap. In the past I have used Angular, Node, Express and others.

Projects

Gigmor

A jobs marketplace connecting musicians and talent bookers. Junior engineer on five-person team working in React.js. Responsible for all markup and styles and contributed to integration on the front-end.

Pokébook

A full-CRUD, Bootsrapped, Ruby on Rails website created in a one-week sprint that allows users to make an account and maintain a list of their favorite Pokémon.

Near Hear

A collaborative Node.js-based site built with the Google Maps and Bands in Town APIs, allowign users to find what concerts are happening near them and get ticket information.

Emojination

A MEAN-stack app that allows users to write stories based on emoji prompts. Featured Materialize formatting that cycled through color schemes randomly with each click and content that was served up with some randomization.

Bright Lights, Big Kitty

A logic-driven card game build with JavaScript, HTML, CSS, and a gratuitous quantity of cute cat pics.
